Case study · chewy

Chewy — The Tracker,
Habit Hacker

A chat-first calorie tracker built around one idea: logging what you eat should be as easy as describing it. This is the end-to-(almost)-end story.

Solo Project By

Mindy Liu

Status

Developing — 04.26

01

Discover

What's the problem?

"When friction is low, habits are easy." — James Clear, Atomic Habits

The observation

Existing calorie trackers like MyFitnessPal require searching databases, logging each ingredient separately, and navigating multiple screens just to record a meal. The friction makes consistency feel like a chore.

The inspiration

I ultimately sought a solution to my own problem — tracking macros consistently. I admired the simplicity built into my favourite apps like Hevy and Goodnotes and thought it could be replicated in the macro-tracking space.

Key insights

> Friends who track macros use AI chatbots, Google Docs and the Notes app, finding macros through online databases and web searches.
> Apps with simple UI interfaces dominate as a method for tracking habitual tasks like notes apps and Hevy.

Friction kills habits Natural language is faster Simplicity drives consistency Good design exists all around us
02

Define

Who are we designing for?

"How might we make logging meals so frictionless that it becomes a natural, automatic part of someone's day?"

Problem statement

People who want to track nutrition struggle to maintain the habit because existing apps demand too much effort per log. The barrier between intention and action is too high. They resort to oversimplistic methods that lack accuracy and depth for historic logging.

Target user

Health-conscious individuals who have tried conventional tracking methods before.

Age demographic: 17–50

Primary use-case: Users logging individual meals within 10 seconds through a single interface, being able to reuse historic inputs and review history to understand patterns.

03

Develop

How are we building it?

Core interaction

A chat-first input bar at the bottom of the screen. Users type a natural language description of their meal. The AI parses it and returns macros. One tap logs it.

Chat-bot Meal Processing Recipe Book Historic Tracking

Major iterations

> Started dark theme → moved to warm light palette to emulate wellbeing essence.
> Removed Create button from main input to reduce cognitive load + added long-press to convert logged meals to recipes.
> Merged tabs into dashboard then combined with profile to streamline progress review.

Dev decisions

Groq AI API — Fast inference layer with free tier and integratable LLM features.
Supabase — Strong backend with built-in verification, databases and storage.
React Native — One codebase for iOS and Android with fast deployment through Expo.

Design decisions

Colours — Warm paper palette instead of clinical white. Accent colour based off apple logo.
Typography — High readability with a typography-led hierarchy.
Format — Minimal tab bar with three clear destinations: Chat, Meals, Profile.

04

Deliver

What are we at now?

Three modes, one interface — Log it, Check it, or turn it into a Recipe.

Final product

A React Native app with three tabs: a chat-first food logger, a recipe library, and a profile with daily goals, meal history, calendar, and a 7-day chart. Backed by Supabase and a custom AI parsing function.

What's next

> Photo-to-log via camera input.
> Audio description mode.
> Meal sharing between users.
> A more refined onboarding that personalises goals.

Reflection

When I first started this project I thought it was a pretty lukewarm idea. The more I looked into the problem, the more realistic it became as a solution. The product hinges on simplicity and naturally my first thought at a solution was in fact the simplest idea. I've learnt a lot from the project — new languages, software tools, where best to design, develop and deploy, and most significantly I've built a skill for product-thinking. There's more to come from this project as I use it to explore LLMs and AI integration, but for now this is what I've chewed up.
